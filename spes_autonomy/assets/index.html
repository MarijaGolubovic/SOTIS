Translate

In the "Translate" section, we'll explore the translation action. This action allows the robot to move to a new position in a specified direction.

The goal of this lesson is that you understand concept of variable and params in programing. The transalte action present simple action who only move robot on specific position, user only send distance who want to them robot travel. And exactly that distance represent variable or parameter in programing.

Here's an example using XML:

    <Translate name="TRANSLATE"
               frame_id="base_link"
               ignore_obstacles="True"
               x="0.5"
               error="error"/>

Demo

The above XML code represents a translation action. In this example, the robot is instructed to translate by 0.5 units along the x-axis.

You can adjust the parameters such as frame_id, ignore_obstacles, and x to achieve the desired translation behavior. Experiment with different values to observe how the robot responds.

Rotate

In the "Rotate" section, we'll explore the rotation action. This action allows the robot to turn in a specified direction for a certain number of degrees.

In this lesson, as well as in the previous one, our goal is to explain the role of parameters and variables in programming. This time, we will do so with the help of the yaw parameter in the Rotate Action.

Here's an example using XML:

            <Move name="ROTATE"
                frame_id="base_link"
                mode="1"
                ignore_obstacles="True"
                x="0"
                y="0"
                yaw="150."    
                error="error"
                reversing="0"/>

Demo

The above XML code represents a rotation action. In this example, the robot is instructed to rotate by 150 degrees in the yaw direction.

You can adjust the parameters such as x, y, and yaw to achieve the desired rotation behavior. Experiment with different values to observe how the robot responds.

Repeat

In the "Repeat" section, we'll explore the repetitive action. This action allows the robot to perform a sequence of tasks multiple times.

From what has been said so far, you can infer that we have some repetitive behavior or cycles. That's exactly what loops do in programming. The code inside a loop repeats as many times as the value of the iterator. In the video attached, you can see a good example of such behavior.

Here's an example using XML:

    <Repeat num_cycles="5">
        <Sequence>
            <!-- Your existing actions go here -->
            <Move name="MoveAction"
                  frame_id="base_link"
                  ignore_obstacles="true"
                  x="0"
                  y="0"
                  yaw="20"
                  error="error"/>
            <!-- End of existing actions -->
        </Sequence>
    </Repeat>

Demo

The above XML code represents a repetitive action. In this example, the robot is instructed to move in the yaw direction by 20 degrees, repeated 5 times.

You can adjust the parameters to fit your specific requirements and experiment with different values to observe how the robot responds.

Fallback

In the "Fallback" section, we'll explore the fallback action. This action allows the robot to execute an alternative action if the primary action fails.

After the introductory sentence, you might already infer which programming concept is being discussed. If the execution of a particular part of the code depends on the fulfillment of a condition, we are referring to an "if" statement or branching. Here, it is represented through a fallback, but the goal is the same. If a certain condition is not met, the current action is interrupted, and the program moves on to the next one.

Here's an example using XML:

    <Fallback>
        <Sequence>
            <Move name="RotateAction"
                  frame_id="base_link"
                  ignore_obstacles="true"
                  x="0"
                  y="0"
                  yaw="90"
                  error="error"/>
            <ForceSuccess>
                <Translate name="MoveForward"
                           frame_id="base_link"
                           ignore_obstacles="false"
                           x="0.3"
                           error="error"/>
            </ForceSuccess>
        </Sequence>
    </Fallback>

Demo

The above XML code represents a fallback action. In this example, the robot is instructed to rotate by 90 degrees. If this rotation fails, it will attempt to move forward by 30 cm.

You can adjust the parameters to fit your specific requirements and experiment with different values to observe how the robot responds.
